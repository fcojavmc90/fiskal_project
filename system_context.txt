Purpose
- Plataforma FISKAL para conectar clientes con profesionales fiscales, con flujo de encuesta, selección de profesional, agenda, pagos y videollamadas.

Core User Flows
- Auth: Login, registro, confirmación de email.
- Perfil: crea UserProfile y (si corresponde) ProfessionalProfile.
- Cliente:
  - Completa encuesta fiscal (20 preguntas + adjuntos).
  - Selecciona profesional (ranking básico por palabras clave).
  - Pago inicial (unlock agenda) vía PayPal.
  - Agenda y reserva con el profesional.
  - Dashboard con citas, casos, mensajes y pagos.
- Profesional:
  - Crea horarios disponibles (agenda).
  - Administra solicitudes de cita y reprogramaciones.
  - Gestiona casos, documentos y chat.
  - Envía links de pago y finaliza casos.
- Videollamadas:
  - Chime meeting creado por API backend.
  - Manejo de reconexión y recreación de reunión en cliente.

UI / Routes (Next.js App Router)
- Login: / (src/app/page.tsx)
- Registro: /register
- Confirmación: /confirm-email
- Encuesta: /survey
- Profesionales: /professionals y /professionals/recommended
- Pago inicial: /checkout
- Agenda: /agenda
- Booking directo: /client/booking?pro=<PRO_SUB>
- Dashboard Cliente: /dashboard-client
- Dashboard Profesional: /expert-dashboard
- Videollamada: /call
- Pago de servicio: /pay?payment=<ID>

Backend / APIs
- GraphQL (AppSync) usado para perfiles, agendas, citas, casos, pagos y mensajes.
- API Chime: src/app/api/chime/meeting/route.ts
  - Soporta fallback a Lambda URL y credenciales directas.

Storage

- Storage requiere Identity Pool válido si se usan credenciales IAM desde frontend.

Payments

- Pagos: unlock inicial + pagos 50/50 por caso
